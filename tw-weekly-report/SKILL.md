# 週報生成系統（/週報）

**吳奇老師個人 Claude 工作系統 V2.0** 週報模組。  
觸發後執行五個標準階段：**掃描 → 分類 → 分析 → 報告 → 更新記憶**。

> 執行前先讀取：
> - `references/role-classifier.md`：四角色分類規則與跨角色機會矩陣
> - `references/report-format.md`：週報視覺化格式規範

---

## 階段一：掃描（Scan）

**立刻執行，無需詢問使用者。**

1. 向使用者說：「正在掃描過去 7 天對話...」
2. 計算 7 天前的 ISO datetime（今天減去 7 天）
3. 調用 `recent_chats` 工具：
   ```
   n=20, sort_order='desc', after=<7天前的datetime>
   ```
4. 若回傳對話少於 3 則，補掃前一週：
   ```
   n=20, sort_order='desc', before=<7天前>, after=<14天前>
   ```
5. 記錄：總對話數、最早日期、最晚日期

---

## 階段二：分類（Classify）

對每則 chat 的標題與摘要，套用 `references/role-classifier.md` 的關鍵詞表分類：

| 角色 | 代號 | 顏色標記 |
|------|------|---------|
| 主編 / 社群管理者 | 🟣 主編 | 紫色 |
| 學術研究者 | 🔵 研究者 | 藍色 |
| 教師 / 教育設計者 | 🟢 教師 | 青色 |
| Vibe Coding 開發者 | 🟩 開發者 | 綠色 |
| 系統/其他 | ⚙️ 系統 | 灰色 |

**分類規則：**
- 一則對話可屬於多個角色（以主要產出物決定主角色）
- 本次「建構工作系統」的對話群歸入⚙️ 系統
- 無法判斷者歸入⚙️ 系統，不強行分類

建立分類結果表：
```
{ role, chat_title, status, key_output, date }
```

其中 `status` 為：
- `done`：對話中任務明確完成（有輸出物）
- `in-progress`：有開始但未完成
- `follow-up`：對話中提到「之後」「待確認」「下次」「TODO」

---

## 階段三：分析（Analyze）

基於分類結果進行三項分析：

### 3.1 生產力統計
各角色：任務總數、已完成數、進行中數、需跟進數

### 3.2 未完成偵測
掃描對話內容中的未完成信號：
- 「之後再做」「待處理」「下次繼續」「TODO」「尚未」「還差」
- 若找到，標記為 `follow-up` 並摘要任務內容

### 3.3 跨角色機會偵測
套用 `references/role-classifier.md` 中的**機會矩陣**，找出：
- 研究者產出 → 是否可轉為主編圖卡？
- 教師案例 → 是否可轉為翻轉教育投稿？
- 開發者工具 → 是否可轉為社群分享素材？

最多列出 **3 個**高品質機會，不強求湊數。

---

## 階段四：生成報告（Report）

調用 `visualize:show_widget` 生成週報視覺化介面。  
詳細格式規範請參考 `references/report-format.md`。

**報告必含區塊：**
1. **週報標題列**：本週日期範圍 + 總對話數
2. **角色工作量分布**：四角色任務數橫條圖
3. **各角色任務清單**：完成 ✅ / 進行中 🔄 / 需跟進 ⚠️
4. **跨角色機會**：最多 3 項，附轉化建議
5. **下週優先任務**：3-5 項，附執行角色標記與難易估計

---

## 階段五：更新記憶（Memory Update）

報告呈現後，問使用者：
> 「以上週報是否準確？下週優先事項是否需要調整？」

確認或使用者微調後：

1. 調用 `memory_user_edits(command='view')` 確認現有記憶
2. 若已有「優先任務」條目 → `command='replace'`（更新）
3. 若無 → `command='add'`（新增）
4. 記憶格式：
   ```
   本週優先任務（YYYY/MM/DD 週報）：①[任務] ②[任務] ③[任務]
   ```
5. 回覆：「✅ 記憶已更新，下週所有對話將自動參考這份優先清單。」

---

## 執行規範

- **全程繁體中文**，語調積極正向
- 若某角色本週零對話 → 標示「本週無相關任務」，不補充假資料
- 優先任務排序原則（高→低）：
  1. 有截止日期的學術任務（碩論、投稿）
  2. 期刊發布週期相關（數位敘事力）
  3. 教學日程（考試前後）
  4. 開發維護需求
- 整份週報執行時間目標：**90 秒內完成**（不含使用者確認等待）
