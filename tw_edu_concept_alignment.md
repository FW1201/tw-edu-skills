# 概念對齊協議（Concept Alignment Protocol）
# 版本：1.0 ｜ 所有 tw-edu-* skills 的前置執行步驟

---

## 為什麼需要概念對齊？

教師說「幫我出一份考卷」，但真正的意圖可能是：
- A. 診斷學生的學習落差
- B. 期末總結性評量
- C. 練習學生的素養應用能力
- D. 用於差異化分組的前測

這四種意圖，對應的題型、難度、格式完全不同。
**不做概念對齊就直接執行 = 高概率做出「技術上正確但方向全錯」的東西。**

---

## 概念對齊的三個維度

```
維度一：目標對齊（Why）
  → 這個任務最終要達成什麼？
  → 成功的樣子是什麼？

維度二：對象對齊（Who）
  → 目標學生的特性（年級/程度/需求/情境）
  → 教師的角色與限制

維度三：脈絡對齊（Context）
  → 這個任務在更大的教學流程中處於什麼位置？
  → 有沒有前提假設需要確認？
```

---

## 概念對齊執行流程（CAP Flow）

```
Step 1：接收輸入
  從使用者的訊息中提取：
  ・明確要求（explicit request）
  ・隱含意圖（implicit intent）
  ・可能的衝突（conflicts / assumptions）

Step 2：鏡像反射（Mirror）
  用自己的語言重述使用者的需求：
  「我理解您想要的是___，目的是___，
   給___年級的學生使用，背景是___。
   請確認我的理解是否正確？」

Step 3：澄清關鍵分歧（Clarify Gaps）
  最多提出 2 個最重要的澄清問題：
  ・只問「答案會影響執行方向」的問題
  ・不問「可以自己合理推斷」的問題

Step 4：確認共識後執行
  使用者確認後，再開始實際工作。
  若使用者說「直接做」，跳過對齊步驟（使用者知情的快速模式）。
```

## 各 Skill 類型的概念對齊重點問題

### 教案類（lesson-plan、differentiated、interdisciplinary）
```
對齊問題集：
A. 「這節課最重要的『學習轉折點』是什麼——
   學生在這節課前後，認知上會有什麼本質的改變？」
B. 「您預期學生會遇到什麼主要困難或迷思概念？」
C. 「這份教案是您第一次教這篇文/主題，
   還是已教過、想改善某個部分？」

對齊用途：
→ A 確認教案的核心驅動
→ B 影響「發展活動」的設計深度
→ C 決定搜尋策略（全新 vs. 局部優化）
```

### 評量類（exam-generator、rubric、formative）
```
對齊問題集：
A. 「這個評量的主要用途是什麼？
   診斷（診斷學習落差）/ 形成（即時回饋）/ 總結（成績評定）？」
B. 「您希望難度偏向哪裡？
   大部分學生應能答對（低通過門檻）
   vs. 區辨高低程度（鑑別度優先）？」
C. 「學生有機會用 AI 工具完成這份評量嗎？
   如有顧慮，是否需要抗 AI 設計？」

對齊用途：
→ A 決定題型比例和難度分布
→ B 影響 ★ 難度標記和布魯姆層次
→ C 是否觸發抗 AI 評量矯正流程
```

### 教材製作類（worksheet、slides-creator）
```
對齊問題集：
A. 「這份教材在教學流程的哪個位置使用？
   課前（預習激活）/ 課中（核心學習）/ 課後（鞏固）？」
B. 「學生會獨立使用，還是在教師引導下使用？」

對齊用途：
→ A 決定鷹架密度和自主程度
→ B 影響說明文字詳細程度
```

### 學生表現類（feedback-writer、learning-portfolio）
```
對齊問題集：
A. 「您希望評語達到什麼效果？
   激勵學生繼續（情意）/ 指出具體改進方向（認知）/ 兩者兼顧？」
B. 「這位學生目前的整體學習狀況如何？
   （讓我能調整評語的語氣和深度）」

對齊用途：
→ A 決定三明治回饋的比例
→ B 避免「好心辦壞事」（例如：對挫折感嚴重的學生給太多負面建議）
```

### 班級行政類（parent-communication、meeting-facilitator）
```
對齊問題集（親師）：
A. 「這封信的主要情緒基調是什麼？
   常規通知（中性）/ 關懷提醒（溫暖）/ 正式記錄（嚴肅）？」
B. 「家長對這個議題是否已有既有立場或情緒？」

對齊問題集（會議）：
A. 「這次會議最希望達成的一個具體決議是什麼？」
B. 「與會者之間對議題有沒有已知的意見分歧？」
```

### 研究類（research-viz、citation-checker）
```
對齊問題集（視覺化）：
A. 「這張圖最終的讀者是誰？
   指導教授（嚴謹）/ 研討會聽眾（清晰）/ 論文審稿人（精確）？」
B. 「圖是用來解釋理論，還是呈現結果數據？」

對齊問題集（文獻查核）：
A. 「這份文獻清單的來源是什麼？
   自己整理 / AI 生成 / 他人提供 / 混合？」
   → 若包含 AI 生成，自動提升查核嚴格度
B. 「最終用途是什麼？
   學位論文 / 期刊投稿 / 課堂報告？」
   → 影響格式驗證嚴格程度
```

## 概念對齊的輸出格式

執行對齊後，Claude 輸出一個「對齊確認卡」：

```
╔══════════════ 概念對齊確認 ══════════════╗

📌 任務：{具體任務名稱}
🎯 核心目標：{使用者真正想達成的目的}
👤 目標對象：{學生特性 / 年級 / 情境}
📐 脈絡：{在教學流程中的位置}
⚠️  前提假設：{Claude 做了哪些推斷，需使用者確認}
❓ 未確認事項：{若有，最多 2 個問題}

確認後我立刻開始。
（若有修改，直接說明；若確認無誤，回覆「OK」或「確認」）
╚═══════════════════════════════════════════╝
```

## 快速模式（跳過對齊）

以下情況可跳過完整對齊，直接執行：
1. 使用者在指令中已提供所有關鍵資訊（年級、科目、目的、格式均清楚）
2. 使用者明確說「直接幫我做」「不用確認」
3. 重複執行相同任務（使用者說「跟上次一樣的格式」）

即使快速模式，Claude 仍應在輸出前確認「我理解的目標是___」。

---

## 對齊失敗的警示信號

以下情況 Claude 應暫停並要求對齊：
1. 任務描述模糊到無法判斷年級或用途
2. 不同部分的指令互相矛盾（例：「簡單一點但要深度」）
3. 輸入的評量內容和聲稱的目標明顯不符
4. 使用者說的和上傳的文件內容不一致
