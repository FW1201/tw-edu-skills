# 教材轉視覺化教學簡報 v1.0

## 哲學定位
「好的教學簡報不是把文字搬上投影片，而是把複雜知識視覺化。」

## Step 1：年級偵測 + 引導式資訊收集

### 第一輪問題（最多 3 個）
```
Q1: 「這份簡報的教學主題或課文是什麼？」
Q2: 「給哪個年級的學生使用？」（若未偵測到）
Q3: 「您有現成的教材想上傳嗎？（Word/PDF/文字說明）」

提示：「您可以直接貼上文字內容、上傳 Word 或 PDF，
        或用文字描述想要什麼內容，我都能製作。」
```

### 第二輪問題（選填）
```
Q4: 「想要什麼教學風格？
    A. 簡約現代（白底藍色系，適合正式課堂）
    B. 活潑彩色（鮮明配色，適合低年級或互動課）
    C. 深色沉穩（深藍/深灰底，適合高中/專業課程）
    D. 台灣特色（融入本土元素）」

Q5: 「幾張投影片？（預設依內容自動判斷，通常 10-20 張）」
```

### 確認摘要
```
✅ 主題：{主題}
🎓 年級：{年級}（→ 字體大小/資訊密度已自動調整）
🎨 風格：{風格選擇}
📊 張數：約 {N} 張（含封面/目錄/每節內容/結尾）
📄 輸出：{主題}_教學簡報.pptx
```

## Step 2：年級對應簡報規格

| 學習階段 | 每頁字數上限 | 圖文比例 | 字體大小 | 動畫建議 |
|---------|------------|---------|---------|---------|
| 國小低年級 | 20字 | 圖 70% | 標題 40pt / 內文 28pt | 多（出現動畫） |
| 國小中高年級 | 40字 | 圖 55% | 標題 36pt / 內文 24pt | 適中 |
| 國中 | 80字 | 圖 40% | 標題 32pt / 內文 22pt | 少（重點出現） |
| 高中 | 120字 | 圖 30% | 標題 28pt / 內文 20pt | 無或極少 |

## Step 3：簡報結構設計

### 標準教學簡報架構（約 15 張）
```
01. 封面（標題 + 科目 + 年級 + 日期）
02. 本節學習目標（3-5個，使用布魯姆動詞）
03. 先備知識激活（提問/圖片/情境引導）
04-12. 主要教學內容（每個概念一張，含視覺化）
13. 重點整理（心智圖或條列）
14. 課堂活動說明（學習任務）
15. 下節預告 + 課後任務
```

### 各類型投影片設計規範

#### 概念說明頁
- 左圖右文或上圖下文配置
- 最多 3 個重點（使用 ▸ 或 ① ② ③）
- 關鍵詞用色塊或底線突出

#### 提問/討論頁
- 大字問題（居中，字體 36pt 以上）
- 思考時間計時器說明（如：「請思考 1 分鐘」）
- 國中以上：提供思考框架（兩欄：「我認為…因為…」）

#### 圖表解讀頁
- 圖表置中/上方，說明文字在下
- 標注關鍵數據（用顏色或箭頭）
- 附一個引導問題

#### 活動說明頁
- 任務步驟清單（Step 1, Step 2...）
- 時間配置說明
- 分組/個人說明

## Step 4：生成精美 pptx

**若使用者上傳了文件或貼入文字，先解析內容：**
```bash
# 解析上傳的文件內容
python3 -m markitdown /mnt/user-data/uploads/[檔案名] 2>/dev/null || \
  echo "[請使用者直接描述內容]"
```

**生成簡報：**
```bash
python scripts/generate_slides.py \
  --title "[主題名稱]" \
  --subject "[科目]" \
  --grade "[年級]" \
  --stage "[elementary/junior/senior]" \
  --style "[modern/colorful/dark/local]" \
  --content_file "[已解析的內容路徑]" \
  --slides_count [張數] \
  --output "/mnt/user-data/outputs/[主題]_教學簡報.pptx"
```

## Step 5：品質確認清單

- [ ] 封面有完整資訊（主題/科目/年級）
- [ ] 學習目標頁使用布魯姆動詞
- [ ] 每頁字數符合年級規格
- [ ] 圖文比例符合年級規格
- [ ] 主色調一致，最多使用 3 種顏色
- [ ] 重要概念有視覺化呈現（圖/表/色塊）
- [ ] 至少一張互動/提問投影片
- [ ] 結尾頁有重點整理

## MCP 連接器（平台差異）

### Claude Code ／ Claude.ai（Pro/Team/Enterprise）
```
Canva MCP（若已連接）：
  - 自動生成高設計感的簡報
  - 支援 Canva 模板套用
  - 比 python-pptx 視覺效果更佳
  - 觸發條件：使用者提到「Canva 風格」或「高設計感」

Google Drive MCP（若已連接）：
  - 直接讀取 Drive 中的教材文件
  - 完成後直接儲存回 Drive
  - 觸發：使用者提供 Google Drive 連結

注意：若 Canva MCP 可用，優先使用 Canva MCP；
      若不可用，則使用 python-pptx 腳本生成。
```

### Claude.ai（啟用方式）
Settings → Connectors → 啟用 Canva + Google Drive

### Canva MCP 調用範例
```
若 Canva MCP 可用時，使用：
generate-design(design_type="presentation", query="...")
而非執行 Python 腳本
```

### 其他平台（Codex / gemini-cli）
MCP 不可用，使用 Python pptx 腳本生成，並提示使用者下載文件
