## AI 可答性的本質分析

AI 擅長的評量類型：
1. **封閉式知識回憶**：定義、列舉、配對
2. **通用格式寫作**：無個人情境的說明文、記敘文
3. **標準化分析**：固定框架的文學分析、段落結構說明
4. **公式化應用**：計算、填表、按步驟操作

AI 弱點（無法偽造）：
1. **真實個人敘事**：你自己昨天、上週、今年發生的事
2. **在地脈絡知識**：你班上的事、你學校的事、你的社區
3. **當下即時反應**：現場口述、即席討論
4. **元認知展示**：說明「你為什麼這樣想」的真實過程
5. **多輪對話中的一致性**：追問後能否維持相同立場並延伸

## 各年段適用策略優先級

### 國小（第一至三學習階段）
優先策略：
- 圖畫 + 口述（AI 無法畫出學生自己的圖）
- 指定真實物品觀察（「描述你帶來的植物」）
- 家人訪談記錄
- 現場表演/展示

較不適合：
- 複雜的元認知要求（認知負荷太高）
- 多步驟文字過程記錄（書寫尚不流暢）

### 國中（第四學習階段）
最有效策略：
- RAFT 寫作框架（個人化讀者設定）
- 本學期閱讀連結（AI 不知道你讀了什麼）
- 班級共同事件引用
- 兩段式：書面 + 口頭說明

### 高中（第五學習階段）
最有效策略：
- 學習歷程整合任務（AI 不知道你的歷程）
- 本地議題分析（AI 對在地資訊掌握有限）
- 與前次作品的連結與成長說明
- 反思日誌（多次累積，AI 難以一致偽造）

## RAFT 框架完整應用指南

**R = Role（角色）**
讓學生扮演真實存在的角色：
- 記者、醫生、設計師、社工（高中）
- 班刊主編、校廣播員、圖書委員（國中）
- 老師的小助理、小導覽員（國小）

**A = Audience（讀者）**
設定真實讀者，不是只有教師：
- 班上同學、學弟妹、家長
- 社區居民、地方政府
- 未來的自己（Time capsule）
- 課文中的人物（創意類）

**F = Format（格式）**
使用非傳統格式，AI 較難保持格式一致性：
- 公告海報文字、社群媒體貼文
- 使用說明書、懶人包
- 電影評論、播客腳本
- 食譜/操作步驟

**T = Topic（主題）**
加入「只有你知道」的條件：
- 「根據你個人的觀察/經驗」
- 「連結我們這週學習的X和Y」
- 「以班上發生的某件事為例」

## 過程文件化的設計模板

### 三稿模式
```
第一稿：初次嘗試（不評分，只給回饋）
  → 教師/同儕給出 2 個具體改進建議

第二稿：根據回饋修改
  → 說明你採納了哪個建議，為什麼
  → 說明你沒有採納哪個建議，為什麼

第三稿：最終版本
  → 附上「從第一稿到第三稿，你學到的一件事」
```

### 反思卡
```
每次提交附上：
1. 完成這個任務最困難的是___
2. 我用了___策略來克服困難
3. 如果重做，我會改變___
4. 我認為我最好的地方是___（具體說明，不接受「很好」）
```

## AI 可答性快速評估 Checklist

教師自我檢查清單：

□ 這個題目能在 ChatGPT 中直接貼入並得到滿分答案嗎？
  → 是 → 需要矯正

□ 答案需要學生本學期課堂上的特定經驗嗎？
  → 否 → 考慮加入本地情境

□ 學生能否直接複製貼上 AI 答案而無需修改？
  → 是 → 加入個人化要求

□ 評量只看成品，不看過程嗎？
  → 是 → 考慮加入過程要求

□ 如果兩個學生交出一模一樣的答案，你能發現嗎？
  → 否 → 評量缺乏個人化

□ 這個題目要求學生說明「為什麼這樣想」嗎？
  → 否 → 加入元認知要求
