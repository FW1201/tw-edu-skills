# 語音與字幕

## 預設語音方案

預設使用 ElevenLabs。
第一次使用時，先引導使用者完成：

- 建立 ElevenLabs API key
- 把 key 存成環境變數 `ELEVENLABS_API_KEY`
- 選擇 voice id
- 視需要指定 model id

建議同時預留這些變數：

- `ELEVENLABS_API_KEY`
- `ELEVENLABS_VOICE_ID`
- `ELEVENLABS_MODEL_ID`

## API 取用原則

- API key 不可寫進前端
- 旁白生成放在伺服器端或本機腳本
- 生成後再把音檔放入 `public/` 或對應資源資料夾
- 如果要公開發布，保留 AI 旁白揭露或說明

## 第一支旁白的最小流程

1. 準備一小段課程口白
2. 呼叫 ElevenLabs TTS
3. 輸出 MP3
4. 轉成可供 Remotion 使用的素材
5. 檢查音量、速度、停頓與結尾收音

## 字幕生成

若已經有旁白音檔，先做轉錄，再做字幕。

建議流程：

- 先把音訊轉成可轉錄的格式
- 再產生 JSON captions
- 再把 captions 對齊到畫面節奏
- 再做逐鏡字幕頁面

## 字幕顯示原則

- 字幕不要太長
- 一次只承擔一個重點
- 保持白底/深底對比足夠
- 避免遮住核心圖解或關鍵數字
- 若要做逐字高亮，先確認閱讀速度不會太快

## 建議的檢查項目

- 口白是否太快
- 字幕是否跟得上語速
- 每句字幕是否太長
- 專有名詞是否需要人工修正
- 旁白停頓是否和鏡頭切換一致
