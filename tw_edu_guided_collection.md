# 引導式資訊收集框架
# Guided Information Collection Framework
# 版本：2.0 ｜ 所有 tw-edu-* skills 共用

---

## 設計原則

**三輪漸進式收集**：不一次問太多，避免使用者放棄。

```
第一輪（30秒能回答）：核心必填（2-3個問題）
  ↓ 收到答案後
第二輪（依情況觸發）：進階選填（1-2個問題）
  ↓ 收到答案後
第三輪（自動推斷）：Claude 依前兩輪自行推斷並確認
  ↓ 使用者確認後
→ 開始執行任務
```

## 各 Skill 啟動問題腳本

### 通用開場（所有 skill 共用）
```
[Claude 的第一句話格式]
「我幫您準備 [任務名稱]。
先確認幾個關鍵資訊（不超過3個問題）：

1. [最重要的問題]
2. [次重要的問題]

（如果上面都清楚了，我馬上開始；其他細節我會依專業判斷填入，您之後可以修改。）」
```

### A1 教案生成
```
必問（缺一不可）：
→「課文名稱或教學主題是什麼？」
→「哪個年級、哪個科目？」

選問（未提供就詢問）：
→「教材版本？（南一/康軒/翰林/龍騰/三民，沒有偏好就說不限）」
→「幾節課？（沒提的話預設3節）」

推斷（不問）：
→ 設計者姓名 → 填「教師」
→ 班級特性 → 填「一般班級」，若需差異化設計請使用者補充
```

### B2 試卷生成
```
必問（缺一不可）：
→「考試範圍（單元/課文名稱）？」
→「哪個年級？」

選問：
→「題型組合？（預設：選擇 40 分、填充 20 分、問答 40 分）」
→「總分是 100 分嗎？」

推斷（不問）：
→ 難度分布 → 依年級自動套用（見年級適應系統）
→ 布魯姆層次 → 依年級自動設定
```

### E1 親師溝通
```
必問：
→「主要要傳達什麼訊息？（一句話說明）」

選問：
→「需要 Line 訊息短版嗎？」
→「需要附回條虛線嗎？」

推斷（不問）：
→ 書信格式 → 自動套用臺灣標準格式
→ 稱謂/結語 → 自動生成標準格式
```

### D1 回饋評語
```
必問：
→「這是哪種作業/任務的評語？（作文/報告/口頭/其他）」
→「幾位學生？請描述每位學生的表現特點。」

選問：
→「用哪些向度評量？（預設依任務類型自動設定）」

推斷（不問）：
→ 等第 → 依學生表現描述自動判定
→ 字數 → 每位約 100-150 字
```

### F（簡報、小程式）
```
必問：
→「想做什麼主題的教材？」
→「給哪個年級的學生使用？」

選問：
→「有沒有現有的資料/文件可以提供？（可上傳）」
→「希望什麼風格？（活潑/正式/簡約）」
```

## 未知資訊的智慧預設值

當使用者沒有提供特定資訊時，Claude 依以下規則自動填入：

| 資訊項目 | 智慧預設值 |
|---------|-----------|
| 教材版本 | 「多版本通用」 |
| 節數 | 3 節（135 分鐘） |
| 設計者 | 「教師」 |
| 班級人數 | 30 人 |
| 難度分布 | 依年級自動套用 |
| 書信日期 | 今天（當前日期） |
| 評量等第 | 4 等第制 |
| 部署環境 | Vercel（教學小程式） |

## 確認前的摘要輸出（執行任何大型任務前必做）

在開始生成文件前，Claude 輸出一個簡短確認摘要：

```
「好的，我整理了以下資訊，確認後我立刻開始：

✅ 任務：[任務名稱]
📚 主題：[課文/主題]
🎓 年級：[年級 + 學習階段]
📄 輸出：[docx/xlsx/pptx]
⏱ 預計：[節數/時間]
[其他關鍵資訊]

有需要修改嗎？（直接說修改哪裡，或回覆「確認開始」）」
```

這個確認步驟**可選**——若使用者說「直接幫我做」或「不用確認」，
則 Claude 跳過確認直接執行。
