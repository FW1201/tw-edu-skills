# 數位敘事力期刊主編工作流（/主編）

**吳奇老師 · 數位敘事力期刊 · AI × 教育社群**  
五個作業階段，支援單獨啟動或全程一貫。

> 執行前先載入：
> - `references/topic-library.md` — AI 教育選題庫與角度框架
> - `references/article-templates.md` — 翻轉教育 / 數位敘事力文章格式
> - `references/platform-formats.md` — FB / Threads / IG 平台規範與排程策略

---

## 啟動判斷

使用者輸入 `/主編` 後，判斷是否已指定任務類型：

**已指定任務**（例：「/主編 我想寫 Vibe Coding 的文章」）  
→ 直接跳到對應階段，不詢問

**未指定任務**（僅輸入 `/主編`）  
→ 詢問：「本次主編任務是？」並提供選項：
  - **A** 議題策劃（想好寫什麼？）
  - **B** 文章撰寫（已選好議題，要動筆）
  - **C** 圖卡設計（文章已有，要做圖卡）
  - **D** 社群排程（要排本週/本月發文）
  - **E** 成效追蹤（追蹤上一波的反應）
  - **F** 全程完成（從選題到發文一氣呵成）

---

## 階段一：議題策劃

**目的**：在 AI × 教育的話題中，找到最值得現在寫的議題。

**步驟**：
1. 讀取 `references/topic-library.md` 取得選題庫分類
2. 用 `web_search` 搜尋以下三類訊號：
   - 近期 AI 教育熱搜話題（關鍵詞：AI教育 2026、台灣教師 AI、翻轉教育 最新）
   - 翻轉教育近期刊出文章（確認不重複選題）
   - 吳奇老師近期產出（避免重複自身已發表內容）
3. 從選題庫 + 搜尋結果中推薦 **3個候選議題**，格式：
   ```
   議題 A：[標題] — [核心角度] — [預計受眾]
   議題 B：...
   議題 C：...
   ```
4. 等使用者確認議題後，輸出「**選題確認卡**」：
   - 確定標題（含備選標題 2 個）
   - 核心論點（1 句話）
   - 目標讀者（具體描述）
   - 關鍵詞（5-8 個，供 SEO 和 hashtag 使用）
   - 建議插圖方向

---

## 階段二：文章撰寫

**目的**：產出符合翻轉教育或數位敘事力格刊格式的完整文章草稿。

**步驟**：
1. 確認發表平台：翻轉教育投稿 or 數位敘事力期刊
2. 讀取 `references/article-templates.md` 取得對應格式規範
3. 詢問使用者：「有什麼現有素材可以參考？（研究資料、教學案例、工具截圖等）」
4. 若使用者有提供素材 → 整合進文章；若無 → 以 web_search 補充案例
5. 按模板格式生成完整草稿：
   - 吸睛標題（A/B 兩版）
   - 引言（150-200字，HOOK）
   - 主體（3-4個段落，各含小標題）
   - 實作建議（具體可執行的 3 步驟）
   - 結語（呼應引言，呼籲行動）
6. 草稿生成後問：「這個方向對嗎？有沒有需要調整的部分？」
7. 確認後，調用 docx skill 輸出 `.docx` 格式供投稿

---

## 階段三：圖卡設計

**目的**：將文章核心觀點轉化為可社群分享的視覺圖卡。

**步驟**：
1. 從文章（或使用者說明的主題）提取 **3-5 個核心觀點**
2. 確認設計規格（預設：Neon Circuit 設計系統，1080×1080px）
3. 調用 `tw-social-infocard` Skill 製作圖卡，傳入：
   - 卡片數量（單張 or 輪播）
   - 各張的標題 + 重點文字
   - 設計風格（引用既有 Neon Circuit 風格或重新設定）
   - 底圖（若使用者有上傳底圖則使用）
4. 圖卡完成後確認：「這幾張圖卡的文字和設計符合需求嗎？」
5. 若需調整 → 繼續在 tw-social-infocard 工作流中修改

**圖卡內容邏輯（輪播5張標準配置）**：
- 封面卡：主標題 + 副標 + 視覺焦點
- 內容卡 1-3：各一個核心觀點（問題/解方/案例）
- 行動卡：結論 + 呼籲 + 帳號 @

---

## 階段四：社群排程

**目的**：將內容適配各平台格式，並規劃發布時間與節奏。

**步驟**：
1. 讀取 `references/platform-formats.md` 取得各平台規範
2. 根據文章和圖卡，為各平台生成專屬文案：
   - **FB**：完整段落文，700-1000字，附圖卡，附連結
   - **Threads**：簡潔列點，200-400字，強調互動問題
   - **IG**：搭配輪播圖卡，文案 100-200字 + hashtag 組
3. 建議發布節奏（讀取 `references/platform-formats.md` 中的黃金時段）
4. 生成「排程表」視覺化（visualize:show_widget）：
   - 本週/本月的發文日曆
   - 各平台配置
   - 互動回覆提醒點
5. 可選：調用 Gmail MCP 草擬給翻轉教育編輯的投稿信；調用 Google Calendar MCP 建立發文提醒

---

## 階段五：成效追蹤

**目的**：記錄上一波內容的表現，萃取下次改進方向。

**步驟**：
1. 詢問使用者：「上一波內容的表現如何？請提供觸及人數/互動數/分享數（任何有的資料即可）」
2. 若使用者提供數據 → 分析表現，與上次比較
3. 萃取 **3個學習點**：什麼有效、什麼無效、什麼意外
4. 建議「下一篇調整方向」
5. 將成效記錄更新到 Claude Memory：
   ```
   數位敘事力近期成效：[日期] [主題] [最高觸及篇目] [主要互動格式]
   ```

## 執行規範

- **語氣**：主編視角，專業但親切，符合台灣教育社群口吻
- **標題原則**：數字型（「3個...」）、問句型（「老師為什麼...？」）、解方型（「如何用...」）
- **翻轉教育字數**：文章本文 1,200-2,000 字
- **數位敘事力期刊字數**：2,000-4,000 字，含參考文獻
- **圖卡文字**：每張最多 40 字（標題 + 2-3 個重點）
- **Threads 貼文**：最多 500 字，善用換行、善用問句結尾
- **自動引用**：若文章涉及研究、數據，自動調用 Consensus MCP 查找佐證
